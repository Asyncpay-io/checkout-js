# asyncpay-checkout-sdk ‚Äî Claude AI Assistant Context

## Project Overview

`@asyncpay/checkout` is a TypeScript SDK that embeds a hosted checkout iframe into any web page, connecting it to the Asyncpay payment orchestration platform. A single integration gives merchants access to Stripe, Paystack, Flutterwave, Fincra, and PayPal ‚Äî with no per-gateway code on the frontend.

**npm:** `@asyncpay/checkout`
**GitHub:** https://github.com/Asyncpay-io/checkout-js
**Current version:** `0.0.38`
**Demo:** https://asyncpay.io/demo

## How the SDK Works

1. **Caller invokes `AsyncpayCheckout(options)`** ‚Äî This is the only public export.
2. **Input validation** ‚Äî `customerEmail`, `customerUUID`, or a full `customer` object must be present. Amount and currency are validated unless a `subscriptionPlanUUID` or `subscriptionPlanLink` is given. `metadata` keys/values must all be strings.
3. **API call** ‚Äî A `POST` to `/v1/sdk/initialize-payment-request` is made with the merchant's `publicKey` in the `Authentication: Bearer` header.
4. **Redirect vs iframe** ‚Äî If `body.data.should_redirect` is true, the page navigates directly. Otherwise, a full-screen fixed overlay iframe is injected into `document.body`.
5. **`postMessage` event bridge** ‚Äî The iframe posts `eventType` messages (`closeIframe`, `showLoader`, `paymentSuccessful`, `redirect`) back to the parent page. The SDK handles these and fires the appropriate callbacks.
6. **Cleanup** ‚Äî `unsetCheckoutSession()` removes the iframe from the DOM and clears the `sessionStorage` lock key.

## Key Options

| Option | Type | Notes |
|---|---|---|
| `publicKey` | `string` | **Required.** Business public key from the Asyncpay dashboard. |
| `amount` | `number \| string` | Required unless `subscriptionPlanUUID`/`subscriptionPlanLink` is set. |
| `currency` | `string` | ISO 4217 3-letter code (e.g. `NGN`, `USD`). |
| `subscriptionPlanUUID` | `string` | Routes checkout to a subscription flow; ignores `amount`/`currency`/`description`. |
| `subscriptionPlanLink` | `string` | Alternative to UUID for subscription routing. |
| `extendSubscriptionBy` | `string \| number` | Number of billing periods to extend an existing subscription. Sent as `extend_subscription_by` (integer). |
| `metadata` | `Record<string, string>` | Sent as `meta_data`. All keys and values must be strings. |
| `savePaymentMethod` | `boolean` | Saves the payment method used for future off-session charges. |
| `reference` | `string` | Merchant-supplied idempotency reference. |
| `environment` | `"prod" \| "dev" \| "local"` | Defaults to `"prod"`. Use `"local"` to hit `http://localhost`. |

## Build & Dist

```bash
npm run build   # runs: rollup --config
```

Produces three bundles:

| File | Format | Use case |
|---|---|---|
| `dist/bundle.js` | CommonJS | Node / legacy bundlers |
| `dist/bundle.esm.js` | ES Module | Modern bundlers (Vite, webpack 5, Rollup) ‚Äî tree-shakeable |
| `dist/bundle.umd.js` | UMD | `<script>` tag in HTML, exposes `window.AsyncpayCheckout` |

The `dist/` directory is **committed to the repository** and published to npm as-is (see `"files": ["dist"]` in `package.json`).

## Race Condition Guard

The SDK uses two locks to prevent multiple simultaneous checkout sessions:
1. `sessionStorage.getItem("asyncpay-checkout-is-in-session")` ‚Äî survives full-page navigation.
2. `document.getElementById("asyncpay-checkout-sdk-wrapper")` ‚Äî detects an already-rendered iframe.

Both checks throw `SDK_ERROR_CHECKOUT_IN_SESSION` (error code `00002`) if a session is already active.

## Known Technical Debt

- **Hand-maintained `types/index.d.ts`** ‚Äî The declaration file is not auto-generated; it must be kept in sync with `src/index.ts` manually after every interface change.
- **`dist/` committed to git** ‚Äî Avoids a build step during `npm publish` but means the `dist/` tree can drift from source if builds are forgotten.
- **No automated tests** ‚Äî `npm test` exits with an error. Consider adding Vitest or Jest unit tests for the validators and the core checkout flow.

## Development Workflow

1. Edit `src/index.ts` and `types/index.d.ts`.
2. Run `npm run build` to update `dist/`.
3. Open `example/test.html` in a browser (set `environment: "local"` and point `http://localhost` to a running Asyncpay API).
4. Bump `version` in `package.json` and update `CHANGELOG.md`.
5. Commit and publish via `npm publish --access public`.

## Best Practices for AI Assistance

1. **Always keep `types/index.d.ts` in sync** with the `AsyncpayCheckoutInterface` in `src/index.ts`. They are not auto-generated.
2. **Run `npm run build` after every source change** ‚Äî the committed `dist/` is what gets published to npm.
3. **Remove debug `console.log` statements** before any merge to `main`.
4. **Preserve the `window.AsyncpayCheckout = AsyncpayCheckout` assignment** at the bottom of `src/index.ts` ‚Äî it is what makes the UMD build work for `<script>` tag consumers.
5. **Do not break the `postMessage` contract** ‚Äî the iframe relies on `eventType` values (`closeIframe`, `showLoader`, `paymentSuccessful`, `redirect`). Changes here must be coordinated with the hosted checkout frontend.
6. **Validate new options** in `src/modules/validators.ts` rather than inline in `index.ts`.
